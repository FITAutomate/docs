The site is organized in top-level areas, for example:

- `FIT`: launchpad and day-to-day links (start at [Welcome](/fit/))
- `Help` (this section): first-run orientation, including `Getting Started`, `Writing`, and `Publishing`
- other areas: Finance, Marketing, Operations, and the rest of the doc set

## How to use this structure

- begin in `Getting Started` when onboarding a new contributor
- use `Writing` while creating or updating pages
- use `Publishing` before sending updates live

This order matches the normal workflow: understand -> edit -> review -> publish.

## Where pages live

Every page on the site comes from a file in the repository. When you edit a file in GitHub, you are updating the source page for the site.
