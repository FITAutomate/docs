# 👮 SOP Library Rules

This folder contains **SOP** policy/standard documents.

## File Naming (Required)

- Prefix: `📚 SOP <NN> —`
- Format: `📚 SOP <NN> — <Title>.md`

Examples:
- `📚 SOP 01 — Account Management & Identity Standards.md`
- `📚 SOP 09 — Change & Release Management.md`

## Minimum Viable SOP Skeleton (Required)

```markdown
| Field | Value |
| --- | --- |
| Title | <Clean title; no "SOP NN —"> |
| Type | <SOP Plan / Standard / Policy> |
| SOP_TARGET | <INTERNAL / PUBLIC_WEB> |
| Visibility | <Internal / Public> |
| Publish Ready | <Yes / No> |
| Redaction Needed | <None / Light / Heavy> |
| Version | <0.1 / 1.0> |
| Owner | <Name> |
| Last Updated | <YYYY-MM-DD> |
| Applies To | <Optional> |

# <Clean title>

## Purpose
## Scope
## Standards
## Procedure
## Roles and Responsibilities
## Exceptions
## Review Cadence
## Changelog
```

## Heading Rule (Required)

- Exactly one H1 per page.
- H1 is placed **below** the metadata table.
- Use H2/H3 structure consistently for TOC.

## SOP Content Rules

- Keep language policy-grade and specific.
- No credentials, internal secrets, or client-sensitive details.
- Images are generally not used in SOPs.
