# Change Requests & Scope — Public Overview

Automation is not "set-and-forget." Tools change, workflows evolve, and what worked last quarter may need tuning. FIT manages changes using a clear, product-style approach so updates don't create chaos.

---

## Who It Is For

Clients who want to understand how changes to automations and knowledge systems are requested and managed after go-live.

---

## What It Includes

### The 3 Change Types

**Bug Fix**
Something is not working as designed. Examples: workflow fails unexpectedly, routing is incorrect, an expected output isn't produced.

**Tuning / Adjustment**
A small, non-breaking change inside the existing design. Examples: update a rule, modify labels, change an assignment path, tweak formatting.

**Enhancement / Scope Change**
A new capability or a material change to behavior, systems, or requirements. Examples: add a new system, add new workflow logic, expand into new departments, introduce new approval steps.

Enhancements require scoping before we touch production.

### How Changes Are Handled

1. You describe what you want to change and why
2. FIT classifies it (bug fix / tuning / enhancement)
3. FIT checks for downstream impacts and risk
4. We agree on approach and timeline
5. We implement, test, and deploy safely
6. We document what changed

---

## What It Is Not

- Unlimited free changes post-delivery
- Real-time push-button updates
- Guaranteed same-day resolution for all change types

---

## How to Submit a Request

Email [https://forwarditthinking.com/contact](https://forwarditthinking.com/contact) or reach out to your FIT contact with:

- What you want to change (plain language)
- Why (the outcome you want)
- Any examples (screenshots, sample inputs/outputs)
- Priority and deadline (if real)

If you're not sure what type it is, just describe the problem—FIT will classify it.
